# Profile Card Component

A responsive and accessible Profile Card built using HTML, CSS, and Vanilla JavaScript. This project focuses on semantic structure, accessibility, and testability using required data-testid attributes.

## 📌 Features

- Displays user name, bio, and avatar
- Shows current time in milliseconds using JavaScript (`Date.now()`)
- Includes social media links (Twitter, LinkedIn, GitHub)
- Lists hobbies and dislikes
- Fully responsive layout (mobile, tablet, desktop)
- Accessible and semantic HTML structure

---

## 🧱 Structure & Semantics

- `<article>` used as the main card container
- `<header>` for name/title
- `<figure>` and `<img>` for avatar
- `<nav>` for social links
- `<section>` for hobbies and dislikes
- `<ul>` and `<li>` for lists

---

## ♿ Accessibility Notes

- Avatar includes meaningful `alt` text
- All links are keyboard accessible
- Focus styles are visible for navigation
- Social links open in a new tab using:
  - `target="_blank"`
  - `rel="noopener noreferrer"`
- Time element updates dynamically using JavaScript
- `aria-live="polite"` used for time updates

---

## 📱 Responsiveness

- Mobile-first design approach
- Uses Flexbox and CSS Grid
- Layout adapts across:
  - 320px (mobile)
  - 768px (tablet)
  - 1024px+ (desktop)

---

## ⚠️ Known Limitations

- No user input or editing functionality
- Avatar uses a placeholder image (not user-uploaded)
- No animations or transitions implemented
